# 应用本体：文件与架构树说明

此目录结构是我们为 `Mathchine` 应用所设计的 **第一版 Android 代码（Jetpack Compose 版本）的基础原型结构**。
我们尽可能剥离了复杂的花里胡哨功能，紧扣“离线数据包读取”、“极简UI”、“LaTeX渲染”这三大核心主线。

你可以将这部分代码复制、带入到 Android Studio 建立的新项目中运行（需要自行配置正确的 `build.gradle.kts` 等构建文件以导入 Compose、Room 组件包）。

---

## 📂 基础代码目录结构

> **路径总起点**: `应用本体/app/src/main/java/com/mathchine/`

```text
com/mathchine/
│
├── MainActivity.kt        <- Entry Point (应用的单 Activity 主入口，包含极简纯导航拦截与绑定逻辑)
│
├── data/                  <- 数据层 (The Repository & DAO Layers)
│   ├── AppDatabase.kt     <- 数据库实体配置及加载控制（Room Database 构建器）
│   ├── PackManager.kt     <- 数据包处理器，将 `.db` 文件载入到系统数据库安全目录
│   ├── QuestionDao.kt     <- Android Room 离线数据的查询操作，极简化到只提供如“从特定分类随机读取1道题”的功能
│   └── QuestionEntity.kt  <- 和我们通过 Python 打的包表结构极其一致的只读结构类
│
├── ui/                    <- 纯 Compose UI 渲染层 (Views)
│   ├── HomeScreen.kt      <- 第一个极简主页，包含所有种类的题目“一键选择列表面板”
│   ├── PracticeScreen.kt  <- 答题场景页面，包含 题干呈现、四个选项按钮和 即时反馈的区域
│   │
│   └── components/
│       └── LatexView.kt   <- 核心难题攻克组件预留：利用 Webview 或者原生组件，渲染纯文本中带有数学符号的代码
│
└── viewmodel/             <- 状态管理及逻辑流层
    └── PracticeViewModel.kt <- 单独且唯一的业务核心层：负责计算“这道题有没有做对？”，“答错了的话是不是跳出红字和解析？”
```

## 📝 详细说明

1. **`LatexView.kt`** 
它是我们的首个“痛点技术”预演封装。我们刻意剥离了它和其他业务的耦合：对于这整个应用而已，任何一个需要展示复杂数学符号的地方，只要调用 `LatexView(latexText = "$1+1=2$")`，它将来不论是挂载 `KaTeX` WebView 脚本还是安卓原生画布，开发者都不需要在调用层做改动。

2. **`PackManager.kt` & `AppDatabase.kt`**
这组文件负责应对我们“免登陆、强离线数据包管理”的特色需求。我们预计开发者用 Python 构建出 `math_pack.db` 后，只要利用 `PackManager` 脚本丢到指定位置，Room 就会基于 `QuestionEntity` 自动反序列化题目直接供程序读用。

3. **`PracticeViewModel.kt`**
所有的关于用户行为和判定（`判断选择对错` / `提取答题日志反馈` 等等）都被包裹在这里，形成“单向数据流 (Unidirectional Data Flow)”。这意味着 `PracticeScreen.kt` 只负责傻瓜式“发面渲染界面”，从而达到极简开发（代码不冗杂）的好处。
