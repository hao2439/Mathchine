# 数学题 App：极简 Android 客户端结构说明

## 📂 整体架构 (基于 Jetpack Compose + Room)

本项目为**极简架构**，避免繁重的企业级解耦方式，但保留了最清晰的数据单向流模型：
- **UI 层**: 纯粹使用 Jetpack Compose 实现的函数式界面。
- **业务/状态层**: ViewModel (管理答题状态、对错反馈)。
- **数据加载层**: Repository + Room DAO (直接读取 SQLite 数学题目数据包)。

## 文件功能说明清单

以下为核心文件及其设计思路说明（为了展示思路，代码通常存放于相对应的包路径内，但为了简化说明直接列出核心类概念）：

### 1. `MainActivity.kt`
界面的主入口，通常这里什么业务都不会写，只是声明启动 Jetpack Compose 环境并配置应用的主题（主题应该剥离一切花哨动画，仅保留黑白/灰的核心色彩）。

### 2. `NavGraph.kt`
非常简易的导航框架，只设置两个页面路径：
- `/home`：主页，选择题目分类
- `/practice/{category}`：练习页，传递选择的分类参数。

### 3. UI 页面层
- `HomeScreen.kt`
  展示分类列表的界面。界面简单至极：一个 Grid 或 List，每个元素是一个带有类别名字（例如："线性代数"）的纯色块。可以加入状态标识（例如已下载多少题）。
- `PracticeScreen.kt`
  具体答题的界面。分为三块：
  1. 上方题干：调用专门处理 LaTeX 的 Composble组件渲染公式。
  2. 中间选项组：4 个包含 LaTeX 的 Button。
  3. 底部操作栏（如果需要“下一题”逻辑）。

### 4. 核心组件：`LatexView.kt`
这是此应用最核心的前端渲染组件。
为了处理强大的 LaTeX 公式（含矩阵等），我们设计将此处直接包裹一层轻量级的实现（如接入 `MathJax` WebView 或者 `noties/Markwon` 插件），对外暴露为一个可以传入 `Latex String` 的简单 `Composable` 函数。

### 5. 业务逻辑与状态：`PracticeViewModel.kt`
维护当前用户“在这个分类下刷到了第几题”。它管理着：
- `currentQuestion` (当前题目对象)。
- `isCorrect` (对错状态标志位)。
- `showExplanation` (是否展示解析面板)。
- 当用户点击选项时，ViewModel 计算对错并将结果同步回 UI。

### 6. 离线存储与拉取：`Database` & `Repository`
- `QuestionEntity.kt`: 数据库实体的定义，它的字段与“数据制作工具”里那个 `.db` 表结构字段一一对应。
- `QuestionDao.kt`: 简单的数据读取接口，提供一句最重要的代码：`@Query("SELECT * FROM questions WHERE category = :category ORDER BY RANDOM() LIMIT 1")` 来实现抽取一道随机题目。
- `PackManager.kt`: 负责应用的第一次冷启动时、或联网点下载时，把那个预先构建好的 `.db` 包拷贝到 Android 应用内置存储目录下的逻辑脚本。
